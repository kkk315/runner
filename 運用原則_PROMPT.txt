# システム・コンテナ運用原則プロンプト

1. システム設定（APIサーバ・実行制御・リソース制限等）は必ずconfig.yamlで管理し、main.py等のコードで直接上書き・ハードコーディングしない。
2. コンテナ内のユーザー・作業ディレクトリ・権限・環境はDockerfileで定義し、main.py等からdocker run時にuserやenv等で上書きしない。
3. main.pyのdocker runは、Dockerfileで定義されたデフォルト（例: USER runner, WORKDIR /home/runner/tmp）をそのまま使う。
4. 設定値や環境を変更したい場合は、必ずconfig.yamlまたはDockerfileを編集し、コード側はそれを参照するだけに徹する。
5. テストやデバッグ時も、config.yaml・Dockerfileの内容を基準にし、main.py等で一時的な上書きは行わない。
6. どちらの設定ファイルにも記載がない内容は、まずどちらに書くべきかを確認してから実装・修正する。

今後の全ての作業・修正はこの原則を必ず参照し、逸脱しないように進めること。
